## 📱 使用流程

### 普通用户使用流程

1. **注册账号**
   - 打开网站
   - 点击"立即注册"
   - 填写用户名、密码、姓名、手机号
   - 提交注册

2. **等待审核**
   - 管理员会收到审核请求
   - 等待管理员通过

3. **登录使用**
   - 审核通过后登录
   - 浏览代码列表
   - 点击代码查看详情
   - 点击"复制到剪贴板"下载代码

4. **使用下载的代码**
   - 打开代码编辑器（VS Code、记事本等）
   - 新建文件
   - 粘贴代码（Ctrl+V）
   - 保存文件

### 管理员使用流程

1. **登录管理后台**
   - 用 admin/admin123 登录
   - 点击"管理后台"

2. **审核用户**
   - 点击"用户管理"标签
   - 看到待审核用户
   - 点击"通过"或"拒绝"

3. **管理权限**
   - 点击用户的"改权限"按钮
   - 输入权限等级：
     - `normal` - 普通用户（1个文件）
     - `vip` - 会员（2个文件）
     - `svip` - 超级会员（无限制）

4. **管理代码**
   - 点击"代码管理"标签
   - 点击"添加代码"
   - 填写代码信息
   - 点击"删除"可删除代码

## 🔧 自定义配置

### 修改端口

编辑 `web/api/server.js`：
```javascript
const PORT = process.env.PORT || 3000; // 改成其他端口，如 8080
```

### 修改下载限制

编辑 `web/api/codes.js`，找到：
```javascript
const maxDownloads = user.level === 'normal' ? 1 : user.level === 'vip' ? 2 : -1;
```

修改数字即可。

## 🆘 常见问题

### Q1: 双击 启动.bat 闪退？
**A:** 可能是 Node.js 未安装
- 下载 Node.js：https://nodejs.org
- 安装后重启电脑
- 再次双击 启动.bat

### Q2: npm install 失败？
**A:** 网络问题
```bash
# 使用国内镜像
npm config set registry https://registry.npmmirror.com
npm install
```

### Q3: 端口 3000 被占用？
**A:** 修改端口或关闭占用程序
```bash
# Windows 查看占用
netstat -ano | findstr :3000

# 修改端口（见上方"修改端口"）
```

### Q4: Vercel 部署后无法访问？
**A:** 检查以下几点
- 确保 `vercel.json` 文件存在
- 确保 `package.json` 文件存在
- 访问时加上 `/app.html`
- 等待 1-2 分钟让部署完成

### Q5: 数据丢失了？
**A:** 内存存储会在重启后清空
- 本地测试：每次重启都会重置
- Vercel 部署：每次部署都会重置
- 解决方案：使用真实数据库

### Q6: 如何添加更多管理员？
**A:** 编辑 `web/api/db.js`
```javascript
this.users = [
  {
    id: 1,
    username: 'admin',
    password: 'admin123',
    role: 'admin',
    ...
  },
  {
    id: 2,
    username: 'admin2',  // 新管理员
    password: 'password',
    role: 'admin',
    level: 'svip',
    status: 'approved'
  }
];
```
