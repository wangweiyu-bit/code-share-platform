# 🌐 无需安装 Node.js 的使用方案

## 方案：直接部署到 Vercel（推荐）

如果你不想在本地安装 Node.js，可以直接部署到 Vercel，在线使用！

---

## 🚀 部署步骤（10分钟）

### 第1步：注册 GitHub 账号

1. 访问：https://github.com
2. 点击 "Sign up"（注册）
3. 填写邮箱、密码、用户名
4. 验证邮箱
5. 完成注册

### 第2步：上传代码到 GitHub

**方式A：使用 GitHub Desktop（最简单）**

1. **下载 GitHub Desktop**
   - 访问：https://desktop.github.com
   - 下载并安装

2. **登录账号**
   - 打开 GitHub Desktop
   - 点击 "Sign in to GitHub.com"
   - 输入你的 GitHub 账号密码

3. **创建仓库**
   - 点击 "File" → "New repository"
   - Repository name: `code-share-platform`
   - Local path: 选择你的 `web` 文件夹的父目录
   - 点击 "Create repository"

4. **上传文件**
   - 将 `web` 文件夹中的所有文件复制到仓库文件夹
   - GitHub Desktop 会自动检测到文件
   - 在左下角输入：`Initial commit`
   - 点击 "Commit to main"
   - 点击 "Publish repository"
   - 取消勾选 "Keep this code private"（保持公开）
   - 点击 "Publish repository"

**方式B：直接在 GitHub 网站上传**

1. **创建仓库**
   - 登录 GitHub
   - 点击右上角 "+" → "New repository"
   - Repository name: `code-share-platform`
   - 选择 "Public"
   - 点击 "Create repository"

2. **上传文件**
   - 点击 "uploading an existing file"
   - 将 `web` 文件夹中的所有文件拖拽到页面
   - 等待上传完成
   - 点击 "Commit changes"

### 第3步：部署到 Vercel

1. **访问 Vercel**
   - 打开：https://vercel.com
   - 点击 "Sign Up"（注册）

2. **用 GitHub 登录**
   - 选择 "Continue with GitHub"
   - 授权 Vercel 访问你的 GitHub

3. **导入项目**
   - 点击 "New Project"
   - 找到 `code-share-platform` 仓库
   - 点击 "Import"

4. **配置项目**
   - Project Name: 保持默认或自定义
   - Framework Preset: 选择 "Other"
   - Root Directory: 保持默认 `./`
   - 不需要修改其他设置
   - 点击 "Deploy"

5. **等待部署**
   - 部署需要 1-2 分钟
   - 看到 "Congratulations!" 表示成功

6. **访问网站**
   - 点击 "Visit" 或复制网址
   - 网址格式：`https://code-share-platform-xxx.vercel.app`
   - 访问：`https://你的网址/app.html`

## ⚠️ 重要提示

### 数据存储

**当前状态：**
- 使用内存存储
- 每次重新部署，数据会重置
- 适合测试和演示

**如果需要数据持久化：**
1. 需要添加真实数据库（MySQL/MongoDB）

**如何更新网站：**
1. 修改本地代码
2. 上传到 GitHub（使用 GitHub Desktop）

## 🔧 自定义域名（可选）

### 使用自己的域名

1. **在 Vercel 中添加域名**
   - 进入项目设置
   - 点击 "Domains"
   - 输入你的域名
   - 按照提示配置 DNS

2. **配置 DNS**
   - 在你的域名提供商处
   - 添加 CNAME 记录
   - 指向 Vercel 提供的地址

## 🆘 常见问题

### Q1: 部署失败？

**检查：**
1. 确保所有文件都上传了
2. 确保 `package.json` 在根目录
3. 确保 `vercel.json` 在根目录
4. 查看 Vercel 的错误日志

### Q2: 访问网站显示 404？

**解决：**
- 确保访问的是 `/app.html`
- 完整网址：`https://你的网址/app.html`

### Q3: 网站可以访问，但功能不正常？

**检查：**
1. 打开浏览器开发者工具（F12）
2. 查看 Console 是否有错误
3. 查看 Network 是否有请求失败

### Q4: 如何删除项目？

**步骤：**
1. 进入 Vercel 项目设置
2. 滚动到底部
3. 点击 "Delete Project"
