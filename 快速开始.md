## 🚀 三种使用方式

### 方式1：本地测试（最快）

```bash
# 1. 进入 web 文件夹
cd web

# 2. 安装依赖
npm install

# 3. 启动服务器
npm start

# 4. 打开浏览器
访问：http://localhost:3000/app.html
```

### 方式2：部署到 Vercel（推荐）

#### 步骤1：上传到 GitHub
1. 在 GitHub 创建新仓库
2. 将 web 文件夹中的所有文件上传

#### 步骤2：连接 Vercel
1. 访问 https://vercel.com
2. 用 GitHub 账号登录
3. 点击 "New Project"
4. 选择你的仓库
5. 点击 "Deploy"

#### 步骤3：访问网站
部署完成后，Vercel 会给你一个网址，如：
`https://your-project.vercel.app/app.html`

### 方式3：部署到其他服务器

将 web 文件夹上传到任何支持 Node.js 的服务器，运行：
```bash
npm install
npm start
```

## 💡 重要提示

### 1. 数据存储
当前使用**内存存储**，服务器重启后数据会丢失。

**生产环境建议：**
- 使用真实数据库（MySQL、MongoDB等）
- 修改 `api/db.js` 文件

### 2. 密码安全
当前密码是明文存储，生产环境应该：
- 使用 bcrypt 加密密码
- 使用 JWT 进行身份验证

### 3. 文件上传
当前通过表单输入代码，可以扩展为：
- 支持文件上传
- 支持拖拽上传
- 支持批量上传

## 🆘 常见问题

### Q1: npm install 失败？
**A:** 检查 Node.js 版本，建议使用 v14 或更高版本

### Q2: 启动后无法访问？
**A:** 检查端口 3000 是否被占用，或修改 `api/server.js` 中的端口

### Q3: Vercel 部署失败？
**A:** 确保 package.json 和 vercel.json 在根目录

### Q4: 数据丢失了？
**A:** 内存存储会在重启后清空，建议使用真实数据库

### Q5: 如何修改管理员密码？
**A:** 编辑 `api/db.js`，修改 users 数组中的 admin 用户
